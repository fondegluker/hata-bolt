# Hata

Веб-приложение для парсинга объектов недвижимости с **eri2.nca.by** (пустующие дома) и
**au.nca.by** (коммерческая недвижимость) и их отображения на интерактивной карте.

## Возможности

- **Парсинг** с антипарсинг-защитой: headless Chromium (Playwright), ротация User-Agent,
  случайные задержки, пакетные паузы, часовые лимиты запросов, экспоненциальный backoff,
  поддержка прокси. Скачиваются оригиналы фотографий.
- **4 профиля агрессивности**: Осторожный (раз в неделю), Сбалансированный (раз в день),
  Ежечасный, Быстрый (раз в полчаса) — переключаются в настройках и на странице парсинга.
- **Информативный процесс**: прогресс-бар, live-журнал, лог-файл, статистика по запускам.
- **Карта** на Leaflet с кластеризацией, цветными маркерами по типу объекта,
  фильтрами (тип / регион / способ продажи), карточкой объекта в drawer и просмотром
  фото в lightbox.
- **Темы**: тёмная (по умолчанию) и светлая.
- **Настройки**: интерфейс, БД, парсинг (профили, прокси, источники), визуализация —
  всё через веб-страницу, без хардкода.
- **Тестовый режим** (mock): генерирует реалистичные объекты и фото локально, без сети —
  для демонстрации и разработки.

## Установка (WSL2, Ubuntu / Debian)

На голой системе достаточно одного скрипта:

```bash
bash install.sh
```

Он установит системные зависимости, PostgreSQL, Python-окружение, браузеры Playwright,
создаст базу `hata`, применит схему и подготовит скрипт запуска.

## Запуск

```bash
./run_hata.sh
```

Откройте http://localhost:5000

При наличии systemd в WSL2:

```bash
sudo systemctl start hata
```

## Первый запуск

1. Откройте **Парсинг**.
2. Отметьте «Тестовый режим (мок-данные, без сети)».
3. Нажмите «Начать парсинг» — БД заполнится демо-объектами с фото.
4. Перейдите на **Карту** — увидите маркеры, фильтры, карточки и lightbox.

Для реального парсинга снимите галку тестового режима и при необходимости
настройте прокси в **Настройках → Парсинг → прокси**.

## Структура

```
app/
  __init__.py        # Flask app factory
  config.py          # настройки (JSON, dot-path get/set)
  db.py              # Postgres connection + schema migration
  houses.py          # запросы к БД
  pages.py           # страницы (dashboard/map/parser/settings)
  api/views.py       # JSON API
  parser/
    engine.py        # Playwright-парсер + mock-генератор
    runtime.py       # фоновый поток, прогресс, логи
  static/            # CSS + JS (Leaflet, map, parser, settings)
  templates/         # Jinja2
config.default.json  # все настройки по умолчанию
install.sh           # установка на голую систему WSL2
run_hata.sh          # запуск (создаётся инсталлятором)
```

## Настройка прокси

Веб-интерфейс → Настройки → Парсинг → прокси:
- `server` — `http://host:port` или `socks5://host:port`
- `username` / `password` — если требуется

## Метод парсинга и антипарсинг

Сервис `eri2.nca.by` защищён nginx-WAF: блокирует не-браузерные запросы (403)
и, вероятно, геофильтрует по IP (только Беларусь). Используется headless Chromium
через Playwright — реальный TLS-отпечаток (JA3), реальный JS-движок, что делает
запросы неотличимыми от человека. Дополнительно: ротация UA, случайные задержки,
пакетные паузы, часовые лимиты, backoff на 403/429, опциональный прокси.
